JavaScript (JS)

JavaScript (JS) is a programming language mostly used to dynamically script webpages on the client side.

Operators

JavaScript operators are used to assign values, compare values, perform arithmetic operations, and more.

Variables

A variable is a named location for storing a value. That way an unpredictable value can be accessed through a predetermined name.

Variable Naming Conventions

JavaScript variables are named using camel-case. The first word should always start lowercase, with every new word capitlized: newVariableName.

Functions

A function is a code snippet that can be called by other code or by itself, or a variable that refers to the function.

Methods

A method (instnace or static) is a function which is a property of an object.

Arguments

An argument is a value (primitive or object) passed as input to a function.

Parameters

A parameter is a named variable passed into a function. Parameter variables are used to import arguments into functions.

Return

The return statement ends function execution and specifies a value to be returned to the function caller.

Chaining Methods

Method chaining is a technique that can be used to simplify code in scenarios that involve calling multiple functions on the same object consecutively.

Strings

The string data type is composed of any number of Unicode characters surrounded by either double or single quotes.

jQuery

jQuery is a JavaScript Library that focuses on simplifying DOM manipulation, AJAX calls, and Event handling.

Booleans

A Boolean is a logical data type that can have only the values true or false.

Undefined

A condition where an expression does not have a correct value, although it is syntactically correct

NaN

The global NaN property is a value representing Not-A-Number.

Escape

The escape() function computes a new string in which certain characters have been replaced by a hexadecimal escape sequence.

Alert

The alert() method displays an alert box with a specified message and an OK button.

Comments

Textual notations within markup. Though hidden in the browsser, comments are available to be read in the source view.
